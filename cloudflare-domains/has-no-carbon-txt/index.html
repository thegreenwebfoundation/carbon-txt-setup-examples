This site has no carbon.txt file served from it

As the title suggests, this domain should not have a carbon.txt file served from it, nor do any delegation of serving the file to another domain.

This file is being served from the cloudflare CDN, and on this domain any path will serve this page. This means that even visiting /carbon.txt/ will serve a 200 OK error in tests.

Why is this here?

Rather than relying on a random domain to check when debugging or testing the behaviour of any carbon.txt validators/checkers, this domain exists to provide a known domain that should never have a carbon.txt file served from it.
